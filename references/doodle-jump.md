# Doodle Jump 类型包（竖版自动弹跳）

> **类型定义**：竖版攀爬弹跳。核心循环一句话：角色自动不停弹跳（无跳跃键），玩家只控制水平方向瞄准更高平台的落点，一屏一屏向上攀升至终点；跌出屏底即败。
> **输入维度**：单轴连续量（水平 ∈ [-1,1]）→ 落 [controls.md](controls.md) **模式 B 连续轴**：陀螺仪为主 + 半屏按压回退，键盘绑同一语义变量。
> 基线来自已验证项目"跳跳速写本"：规则框架来源 Doodle Jump（Lima Sky, 2009；模型内部知识，未联网核实——项目 GDD 如实声明）；原版为无尽制，本作设终点（1000m）以获得明确 WIN 判定，属有意的规则改造。全部硬数值以红线算术 + 固定种子机器人实证背书。

## 设计基线（60fps，px/s、px/s²；10px = 1m 显示换算）

| 参数 | 值 | 来源与理由 |
|---|---|---|
| 重力 GRAV | 2000 | 平台跳跃已验证基线 2400 下调：涂鸦跳原版节奏更"飘"、滞空更长；数值由红线 1 反推校验 |
| 弹跳初速 BOUNCE_V | -950（恒定、无跳跃键，类型铁律） | 跳高 950²/4000 ≈ 225px ≈ 可见世界高 800px 的 28%，一屏可见 2~3 个候选平台，符合原版观感 |
| 水平速度 MOVE_V | 400（空中恒速上限，加速度 3000，无输入减速 1400） | 单跳滞空 0.95s × 400 = 380px ≈ 0.84 屏宽，与原版"一跳横穿大半屏"手感一致 |
| 最大下落速度 MAXFALL | 1100 | 防穿透（承平台跳跃基线思路） |
| 屏幕环绕 | 玩家出左进右/出右进左；平台不环绕 | 原版标志性规则，保留 |
| 相机 | 只上不下 `camTop = min(camTop, py − 0.45×WH)`；死亡线 = 屏底 +40px | 原版规则 |
| 平台 | 宽 64~104、厚 16；静止 / 水平移动(v 60~110) / 易碎 三类 | 尺寸带见红线 3；类型为原版三种基础台 |
| 碎板 | 踩上即碎、**不提供弹跳**，玩家继续下落 | 原版 brown platform 行为 |
| 弹簧 | 踩上 vy=-1500 → 跳高 562px ≈ 2.5× 普通跳 | 原版道具，加速节奏与奖励；强度上限由红线 6 算术反推 |
| 怪物 | 水平往返悬浮巡逻；侧碰即败；上方踩踏可消灭并弹起（+200 分） | 侧碰=原版；踩踏承平台跳跃踩敌惯例（原版射击消灭，本类型免射击键简化——GDD 声明偏差） |
| 收集物 | 碰捡 +100 分 | 主题收集物 |
| 计分 | 高度 1 分/m + 收集 100 + 踩怪 200 + 通关 1000 | — |
| 生命 | 无生命制，坠落/碰怪即终局 | 原版规则 |
| 终点 | 1000m（世界 10000px，GOAL_Y=-10000）大平台，站上即 WIN | 原版无尽制的有意改造（验收需明确 WIN）；高度由红线 5 时长预算定 |
| 角色 | 碰撞盒 28×80，绘制高 88px（两足竖高基线，≈平台宽 0.9 倍），锚脚底中心、朝向随 vx 翻转 | 与原版角色/平台比例一致 |
| 随机 | 固定种子 mulberry32（seed 写死）→ 每局布局完全一致 | 机器人结果可复现的前提 |

## 可行性红线（违反即不可通关/必死；推导基础：跳高 225px、单跳滞空 0.95s、单跳水平位移 380px）

1. **垂直间距 ≤150**：相邻主台 dy < 跳高 ×0.7（225×0.7≈157 取 150，留 30% 余量——起跳点不可能永远正中最优）；生成器 dy ∈ [70,150]。
2. **水平间距 ≤260**：相邻主台中心水平距离（环绕最短路径）≤ 理论位移 380 × 0.7 ≈ 260；任何布局不得依赖"满速满跳 + 正中起跳"才能通过。
3. **平台宽度 ≥64**：瞄准时间窗 = 台宽/水平速度 = 64/400 = 0.16s ≈ 10 帧 > 人类反应下限 0.1s；移动台 ≥84（额外提前量）；<50 禁用。
4. **主干链必须非碎**：碎板不弹跳，逐台可达主干只能由非碎板构成——生成器以最近非碎板为锚，碎板只做锚点上方的额外插板、永不进主干（初版"只禁连续碎板"被实测证伪：碎板在主干上、下一主台距前一非碎板 >225 即死局，-5013 卡死日志为证）；踩碎板后回落到下方非碎板（间距 ≤150）必可生还。
5. **通关时长 ≈2 分钟**：净爬升 ≈ 平均间距 110px / 0.95s ≈ 115px/s → 10000px ≈ 87s，加失误与移动台等待余量 ×1.4 ≈ 2 分钟；再高一倍则坠落惩罚过重，减半则难度曲线展不开。
6. **弹簧强度上限**：相机只上不下，可生还坠落 = WH − 0.45×WH + 40 = 480px；弹簧跳高必须 < 480 + 最大台距 150 = 630px（初版 -1750/765px 实测坠亡，轨迹日志为证）→ SPRING_V=-1500、跳高 562px，最坏情况（下一台 210 外）回落 352px < 480。
7. **怪物不得封路**：怪物巡逻覆盖带与任一非碎台相交、且怪体底部距台面 <100px 时该台实际不可落（初版 -9444 机器人必死，日志为证）→ 生成器每只怪最多重摇 4 次位置，全冲突则不生成。

**连通性总保证**：生成器从起点逐台向上，每台相对上一主台强制满足红线 1/2/4 → 数学归纳保证起点到终点存在逐台可达路径；机器人实测兜底。

## 实现模式

**数据表示**：无瓦片网格，实体表——`platforms[]: {x,y,w,type(0静止/1移动/2碎),vx,broken,goal}` 按 y 升序生成；`monsters[]: {x,y,x0,range,dir,spd,alive}`；`pencils[]`；`springs` 挂在平台 `{pl,dx}`；`player: {x,y,w=28,h=80,vx,vy,face,squash}`。生成用固定种子 mulberry32，每局布局一致。

**平台生成（两遍）**：第一遍逐台向上——锚点 = 最近非碎板，dy/台宽/类型概率按高度四区难度带取，水平 cx = 锚中心 ±260 内取值后 clamp 进边界（只有玩家环绕）；碎板只置标记不更新锚点；起点平台下方放 2 块安全网宽台；终点大平台（宽 220）贴红线 2 放置。第二遍放怪物——候选点在平台链完整后统一做冲突检查（红线 7）。

**相机与死亡线**：`camTop = Math.min(camTop, p.y - 0.45*WH)`——只取 min 永不下移；`p.y - camTop > WH + 40` 判坠落。

**环绕**：玩家 `x > VIEW_W → x -= VIEW_W`、`x + w < 0 → x += VIEW_W`；所有水平距离判断一律走 `wrapDist`（环绕最短带符号距离）；渲染**环绕双画**——玩家近边缘 60px 时在另一侧 ±VIEW_W 多画一份。

**单向平台**：仅下落（vy>0）且 `prevBottom <= 台顶 + 6` 才着陆；碎板判定通过 → 置 broken、播碎裂、continue 不弹跳。

**物理与输入合成**：`move = |tilt| > |keyMove| ? tilt : keyMove`；水平 vx 以加速度 3000 逼近 ±400、无输入 1400 减速；垂直 `vy = min(MAXFALL, vy + GRAV*dt)`；squash & stretch（bounce 时 -0.28，`squash *= 0.001^dt` 回弹）补偿无奔跑序列帧的动感。

**容易踩的坑**：
1. 弹簧过强 → 峰值回落先触死亡线（红线 6，初版实测坠亡）；
2. 碎板进主干链 → 不可通关死局（红线 4）；
3. 怪物贴台封路（红线 7）；
4. 机器人每帧重选目标 → limit cycle 往返坠落：必须"落台反弹帧一次决策、整段飞行盯目标中心"，跌过目标 218px 才重瞄；
5. 候选台 dy 从头顶量 → 把不可达台纳入候选：必须从**脚底**量，与间距红线同一定义；
6. 移动平台 clamp 边界反弹、玩家才环绕，两者别混；
7. 碎板判定后 `continue` 而不是 `break`——同帧还要能落到下一块台。

## 机器人

可选逻辑快测 `scripts/bot_doodle.js`（用 DOM/canvas 桩在 Node 中执行游戏逻辑）：

- 跑法：`node scripts/bot_doodle.js <index.html> [最长秒数=240]`；连跑 3 次全 WIN 且输出一致才算该长循环快测通过，但不能替代真实 Chromium 合同验证或单独报 `GAME_READY`。
- 决策：近怪避让（|dx|<75 且 |dy|<120 反向躲，踩头态势除外）→ 反弹帧（vy 由正转负）选目标：候选 = 脚底上方 dy∈[40,210] 非碎板，代价 = 2×环绕水平距 + 0.3×dy + 碎板 500 − 终点 1000；跌过目标 218px 重瞄，候选扩到下方 300px 救场；30 秒无新高判卡死。
- **WIN 判定**：`G.state === 'win'`（站上 10000px 终点平台触发），不接受"达到一定时间/高度"等模糊条件。
- **死局处理**：坠落/碰怪 → gameover → 退出码 1；**失败先改游戏**（参数/生成器/判定），不改机器人决策逻辑。

## 表现与反馈（本类型必做）

- 弹跳 squash & stretch（起跳拉长、落台压扁、随 vy 微调）；落台弹跳音。
- 弹簧：专属音效 + 压弹动画 + 粒子；拾取：粒子 + 双连音；踩怪：粒子 + 音效 + 弹起；碎板：碎裂粒子 + 音效；通关：粒子爆发 + 胜利琶音。
- HUD：高度 m、分数、收集数 chip + 总高度进度条；轻量加载页完成后自动开局；WIN/败局面板含星级评价。
- 背景视差（bg 按 -camTop×0.4 滚动）；世界按视野裁剪绘制；竖屏锁定 540×960（ZOOM 1.2、VIEW_W 450、WH 800），横屏窗口居中留边。
- **Sprite 状态结论**：角色没有地面奔跑动作，因此不设计 run 序列；以弹跳、朝向翻转和 squash & stretch 表达 move 状态。这是按动作状态推导的正常结果，不是通用规则豁免。
