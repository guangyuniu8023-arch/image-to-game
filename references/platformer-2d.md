# 2D 横版平台跳跃类型包

> **文件身份**：本文件是横版平台跳跃的已验证类型基线，不是项目 GDD，也不是通用模板。所有项目先按 [gdd-strategy.md](gdd-strategy.md) 生成自己的 `GDD.md`；本包只提供可继承的类型结论和实现约束，禁止跨类型照抄数值。

使用平台跳跃类型时，写代码前按本基线定稿。括弧内是已验证可通关的基线值（来自"汪汪大冒险"：小狗主角、骨头收集、紫色小怪兽、旗杆+狗屋终点），直接沿用或微调。

## 主题设定

- 主角：用户图片主体（抠成透明 PNG，裁到内容包围盒）。**绘制尺寸按体型类别校准**：
  - 四足/横宽主角（狗、猫等）：绘制高度约 62px，碰撞盒 34×50
  - 两足/竖高主角（人等）：绘制高度约 88px（≈2 格），碰撞盒约 34×84
  - 换体型类别必须重新校准这两个数，并截图目检"主角 ≈2 格、与砖块比例自然"；标题页/结算页等展示位一律等比绘制，禁止写死宽高硬拉伸
- 收集物：贴合主题（狗→骨头 / 猫→鱼干 / 兔→胡萝卜 / 默认→星星）
- 敌人：一种巡逻小怪即可（可踩扁，侧面碰伤）
- 终点：旗杆 + 主题建筑（狗屋/城堡/家）
- 标题与文案：中文、可爱风（如"汪汪大冒险"）

## 机制与物理参数（60fps，单位 px/s、px/s²）

| 参数 | 基线值 | 说明 |
|---|---|---|
| 重力 GRAV | 2400 | 与跳跃初速成对决定跳高 ≈154px（红线 1 反推的地基）；汪汪基线 |
| 跳跃初速 | -860 | 跳高 vy²/2g ≈ 154px ≈ 3.4 格 → 台阶 ≤3 格；改它必须连带改红线 |
| 最大跑速 | 330 | 地面加速度 2600，空中 1700，地面摩擦 2200 |
| 最大下落速度 | 980 | 防止单帧位移超 TILE 穿墙（配合分轴碰撞） |
| 可变跳高 | 松键截断 vy×=0.55：放 keyup 事件执行一次；update 轮询必须带 vy<-220 阈值，严禁 vy<0 每帧连乘 | 点按小跳、长按大跳 |
| 土狼时间 | 0.09s | 离开平台边缘仍可跳；宽容窗口经典量级 0.08~0.12s |
| 跳跃缓冲 | 0.12s | 落地前按跳也生效；与土狼时间同属"手感宽容"两件套 |
| 踩敌判定 | vy>120 且脚底-敌顶 < 22px | 成功后玩家弹起 vy=-520 |
| 受伤无敌 | 1.5s 闪烁 | 给玩家脱离危险区的时长；过短会连锁掉命 |
| 生命 | 3 | 掉坑/侧面碰敌 -1；经典马里奥制，与受伤无敌 1.5s 配套 |
| 计分 | 收集 +100，踩敌 +200，通关 +1000 | 经典量级；通关分 ≫ 过程分，鼓励"到达"而非"刷分" |

## 关卡设计红线（物理可行性，违反必导致不可通关）

- 跳跃最大高度 ≈ vy²/(2g) = 860²/4800 ≈ **154px ≈ 3.4 格** → 台阶/砖块高差 ≤ 3 格
- 跳跃最远距离 ≈ (2×vy/g)×跑速 ≈ 0.72s×330 ≈ **236px ≈ 5.2 格** → 缺口 ≤ 4 格（稳妥用 3 格）
- 顶得到的砖块：其底边距站立面 ≤ 200px（玩家跳起头顶可达约 250px，留余量）
- 检查点重生点、敌人出生点：逐个确认正下方有地面
- 敌人不要放在台阶实体内部、单向浮台正上方边缘、唯一通路的死角

## 关卡分段配方（总长 200~220 列 × 12 行，TILE=45，第 10-11 行为地面层）

1. 起步安全区（~16 列平地，3-5 个收集物教学）
2. 单砖教学（1 个 ? 砖）
3. 首个缺口（3 格）
4. 水管阵（高 2 格、3 格交替，中间放 1-2 个敌人）
5. 砖块群（砖/? 交替 5 连，上方一排收集物）
6. 缺口 + 收集物弧线（弧线即跳跃轨迹暗示）
7. 台阶金字塔（上 4 级下 4 级）
8. 单向浮台区（2 块浮台 + 高处收集物）
9. 检查点（约 60% 处）
10. 混合强化段（砖+敌+缺口组合 2 组）
11. 终点：8 级大台阶 → 旗杆（触旗通关）→ 主题建筑
- 敌人总数 12~18 个；收集物 50~70 个
- 难度曲线：每段只引入一个新元素，组合放后半程
- **配方即断言**：以上数量与组件（?砖 ≥4、砖块群 ≥5、水管 ≥2、台阶 ≥12、浮台 ≥2、敌人 12~18、收集物 50~70、总长 ≥200 列）不是建议，是 `scripts/bot_platformer.js` 完整性断言的硬门槛——“能通关但缺机制”同样判 FAIL（见 verification.md 第 2 级）。

## 已验证变体（登记规则见 new-type.md"变体蒸馏"）

### 二段跳（机制变体，2026-07-19 验证）

- **delta**：空中可再跳 1 次（合计 2 跳），二段力量同首跳（vy=-860）；落地 `jumps` 归零重置；走出平台边缘未跳也可触发那次空跳。实现两处：`press.jump` 分支加 `else if (buffer>0 && jumps<2)`、`moveY` 后 `if (onGround) jumps=0`。
- **红线重推**（若关卡为二段跳设计，上限全部放宽；基线关卡不改动仍可通关）：
  - 台阶 ≤6 格（实测二段顶点 293.7px ≈ 6.5 格，理论 2×154=308）；
  - 缺口 ≤9 格（滞空 ≈2×0.72s=1.44s × 330 ≈ 475px ≈ 10.5 格，留余量）；
  - 顶砖砖底 ≤340px（头顶可达 ≈ 250+147 ≈ 397px 留余量）；
  - **变体陷阱**：可变跳高截断标记（jumpCut）必须每跳独立重置——不重置则二段跳被上一跳的截断状态秒掐。
- **验证证据**：专项脚本实测单跳 147.0px / 二段 293.7px（理论 154/308，PASS）；`bot_platformer.js` 回归 `WIN: true` + 完整性断言全过（机器人只用单跳，基线关卡兼容）；标题页截图验收正常。
