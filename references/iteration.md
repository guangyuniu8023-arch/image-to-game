## 五步流程

1. **恢复项目**：从持久工作区（备份目录）复制代码回临时工作区；用户要"回到上一版"时用环境版本工具的 rollback（无则恢复对应备份）。
2. **复现定位**：用户描述体感问题（"跳不起来""卡墙"）→ 对照下方症状表翻译成代码位置；视觉问题用 `make_ff_pages.py` 快进截图复现。
3. **最小修改**：只动相关代码，不顺手重构。
4. **回归验证**：`node scripts/bot_harness.js index.html` 必须 `WIN: true`；视觉改动再截图对比。
5. **存新版本**：按通用原则第 5 条"交付 → 快照 → 备份"（路径与工具随环境，见 SKILL.md 环境适配表）。

## 症状 → 病因 → 修法 对照表

| 症状 | 病因 | 修法 |
|---|---|---|
| 跳不起来 / 跳太高 | JUMP_V 与 GRAV 不匹配 | 见"调物理参数的纪律" |
| 角色卡进墙/地里 | 碰撞没分轴解算，或单帧位移超过 TILE | 先 X 后 Y 分轴；限制 vy 上限（980） |
| 穿过单向浮台 | 缺少 prevBottom 判断 | 只在 `prevBottom <= 板顶+10` 且下落时生效 |
| 顶不到 ? 砖 | 砖块底边超过头顶可达高度（约 250px） | 砖块下移，遵守 GDD 红线 |
| 某处必死/过不去 | 缺口 > 4 格、台阶 > 3 格、重生点在坑上 | 按机器人死亡列号定位，改地形 |
| 敌人卡死/抖动 | 出生在实体内部，或边缘探测失效 | 出生点下方须有地面；检查脚下方块探测 |
| 受伤后一直死 | 无敌时间 inv 没生效或太短 | 受伤后 inv=1.5s 且碰撞前检查 |
| 画面闪烁/撕裂 | 相机没取整或每帧修改画布尺寸 | 相机平滑跟随；画布尺寸只在 resize 时改 |
| 素材白图 | 路径错、跨域、或没写回退 | 检查文件名；每处绘制都有 imgReady 回退 |
| 某素材显示但变形 | 没保持原图宽高比 | `h = w × (naturalHeight/naturalWidth)` |
| 竖屏显示不全/错位 | 写死了 W 而非 VIEW_W | 相机、裁剪、碰撞相关宽度全用 VIEW_W |
| 旋转屏幕后画面崩 | 没监听 resize 重算布局 | `addEventListener("resize", layout)` |
| 没声音 | AudioContext 在首次用户手势前创建 | 首次按键/点击时再 new |
| 手机按钮没反应 | 只绑了 click，或没 preventDefault | pointerdown/up/leave/cancel 全绑 |
| 预览 404 | 项目放在临时目录，或 index.html 不在交付目录根部 | 移到持久交付目录（index.html 在根）重新发版 |
| 注入的测试代码没运行 | HTML 缺 `</body>` 闭合 | 补闭合标签；注入前断言目标串存在 |

## 调物理参数的纪律

跳跃能力是关卡设计的地基，改物理必须连带检查关卡红线，并重跑机器人：

- 跳跃高度 ≈ JUMP_V²/(2×GRAV)（基线 860²/4800 ≈ 154px ≈ 3.4 格 → 台阶 ≤ 3 格）
- 跳跃距离 ≈ (2×JUMP_V/GRAV) × MAX_RUN（基线 ≈ 236px ≈ 5.2 格 → 缺口 ≤ 4 格）
- 手感飘：GRAV 与 JUMP_V **成比例**上调（如 ×1.2），跳跃高度不变、滞空变短
- 跳得更高：只加 JUMP_V，同时检查高处砖块是否还可顶到、缺口红线是否变化

## 换素材的纪律

- 只换单个素材：走 assets.md 的单品流程（生成 → clean_sprite.py 清理 → 复制进项目），代码零改动（回退注入自动生效）。
- 换主角图：remove_bg.py 重抠，确认 drawImage 的宽高比代码用 naturalHeight/naturalWidth 自适应。
- 换全套主题：重新走一遍素材流水线，但**不动关卡数据和物理**——换皮不改玩法。
