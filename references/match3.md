# 消消乐类型包（经典 match-3）

> **类型定义**：回合制棋盘消除益智。核心循环一句话：玩家每步交换一对相邻棋子制造 ≥3 连线，消除 → 掉落补充 → 连锁追分，限步内达到目标分获胜。
> **输入维度**：点选/拖拽离散对象（棋子）→ 落 [controls.md](controls.md) **模式 C 直指**，无虚拟控件，点选与拖动走同一代码路径。
> 基线来自已验证项目"星光少女消消乐"：规则采用经典 match-3 基线（开心消消乐式；来源：模型内部知识，未联网核实——项目 GDD 如实声明），硬数值以红线推导 + 机器人实证背书。

| 参数 | 基线值 | 来源与理由 |

| 色数 | 6 | 经典基线；6 色下死盘低频但必现，由洗牌机制兜住（星光少女项目实证） |

| 计分 | 普通消除 20/子；星光束清除 10/子；星核清除 15/子；整次消除总分 × 连锁倍率（1,2,3…）；胜利时剩余步数 ×100 | 经典量级；连锁乘算是追分主引擎，代码 val 表一致 |

| 相位时长 | swap/revert 0.16s、clear 0.32s、shuffle 0.6s、fall 以补间收敛为准（1.2s 超时兜底） | 代码实测（index.html update 相位机） |

## 可行性红线

> 来源说明：红线 1/2/3 来自“星光少女消消乐”的目标分、难度带和死盘实测，红线 4/5 从已验证代码的硬约束提炼；本包已把必要证据写全，不依赖外部 GDD 或示例。

1. **目标分红线**：目标分 ≤ 步数 × 实测单步均值 × 0.7。实测单步均值 150~200 → 30 步目标 ≤ 3150~4200；本作取 3000 贴下限，普通玩家可达。
2. **难度带红线**：目标分设在机器人上限的 1/3 以内（贪心机器人显著强于普通玩家；7 连胜证明上限远高于目标）。
3. **死盘必现红线**：有限棋盘 + 随机补充 ⇒ 死盘必然出现。洗牌必须数学保证"无现成消除 + 至少一步可走"（≤200 次 Fisher-Yates 尝试，失败兜底重建棋面），禁止"只重排不验证"。
4. **连锁收敛红线**：连锁模拟设迭代上限（代码 12 轮封顶）+ fall 相位超时兜底（1.2s），否则极端连锁/补间抖动会卡死相位机。
5. **初始棋盘不变量**：无现成消除（生成时禁横/纵三连）且至少一步可走（≤200 次尝试）——机器人开局即断言这两条。

## 实现模式

**数据结构**：`board[r][c] = {t: 类型 0~5, s: null|"row"|"col"|"color", x, y: 浮点绘制坐标, id}`。逻辑坐标 (r,c) 与绘制坐标 (x,y) 分离：所有位移都是改目标格 + 指数趋近补间（`p.x += (c-p.x)*min(1,dt*13)`），绝不逐帧改格子索引。星核用 `t=-1` 表示（无底色），匹配扫描与找步都要特判。

**匹配检测**：findMatches 扫横/纵同色 run ≥3；clusterRuns 把共享格子的 run 合并成连通簇（L/T 交叉判定的前提）；特殊棋子由簇决定——簇格数 ≥5 且（最长 run ≥5 或 run 数 >1）→ 星核，最长 run ==4 → 按方向给星光束；变身落点优先本次交换目标格、其次簇中点，**变身格不计入本次消除**。

**连锁结算（相位机）**：`idle → swap（播完才判定）→ 无效 revert 弹回 / 有效扣步进 enterClear → clear（0.32s 播粒子飘字）→ finishClear（列内下落 + 顶部补充，新子出生 y 用负数从棋盘上方自然落入）→ fall（补间收敛后回 enterClear 再检测，连锁数 +1）→ …→ 无消除进 afterResolve（胜负判定 → 死盘检测 → 回 idle）`。星核交换走 pendingClear 预清除集（双星核清全屏）。传播清除（束/核触发）一律入 clearSet（Map key 去重）。

**死盘处理**：findAllMoves 试交换全部相邻对——只检查两个交换位的 runThrough（经过该格的最长横/纵同色），不必全盘重扫；checkDeadlock 无步 → doShuffle（横幅 + 音效 + shuffle 相位）。

**机器人选步（游戏内与测试共用）**：simSwapScore 在纯数值数组上完整模拟一次交换的连锁收益（同规则的簇/特殊/传播/随机补充，12 轮封顶），botPickMove 贪心选收益最大、同分随机。

**容易踩的坑**：
1. L/T 交叉不合并簇 → 5 格 L 形被误判成两个 3 连，星核生不出来；
2. 变身格计入本次消除 → 特殊棋子刚生成就被自己消掉；
3. 传播清除不去重 → 重复计分/重复入队；
4. 无效交换立即弹回（不等动画播完）→ 视觉跳变；
5. 弹回扣步/计分 → 违反经典规则，机器人专项断言就是抓这个；
6. 消除计分必须按 cause 区分（match/line/color 分值不同），别按格数一刀切。

## 机器人

配套脚本 `scripts/bot_match3.js`（桩 canvas/ctx/Image 后拼接游戏内联 script，直接用 Node 执行）：

- 跑法：`node scripts/bot_match3.js <index.html>`；退出码 0 = 全部通过。
- 流程：初始棋盘不变量断言 → 非法交换测试（被接受播动画、弹回、不扣步、不计分、棋盘复原）→ 贪心 bot 连续走步至终局（每步断言分数不倒退、相位回 idle、20000 帧防卡死）→ 死盘测试（4 组公式构造无步棋盘，断言 checkDeadlock 触发洗牌、洗后无现成消除且有步可走）。
- **WIN 判定**：`G.state === "win"` 且用步 ≤ MOVES_MAX（30）；步数尽未达标 = lose = 退出码 1。
- **注意**：含随机源（同分随机、补充随机），单次结果非确定 → 用连胜次数背书：本包基线 **7 连跑全 WIN**。

## 截图状态

项目为截图固定随机种子并提供同步步进或 `?testState=` 等价入口，至少覆盖：标题、稳定棋盘、交换/消除、连锁或特殊棋子、洗牌提示、WIN/LOSE、竖屏和横屏窗口。不能用随机等待碰到连锁状态。

## 表现与反馈（本类型必做）

- 选中脉动白框；交换/弹回补间；消除缩放淡出 + 星光粒子；掉落加速补间。
- 得分飘字（+N，定位在消除簇重心）；**连锁横幅（×2 起）**；特殊棋子生成飘字（星光束！/星核！）；洗牌横幅；星级结算面板。
- 角色看板娘：呼吸浮动常驻；消除时弹跳、幅度随连锁升高；台词气泡按连锁分档（连锁越高越兴奋）；胜利持续欢呼——角色图驱动的核心情感反馈。
- 音效（WebAudio 合成，可静音）：选中/交换/弹回/消除（音高随连锁抬升）/线消/色消/掉落/洗牌/胜/负。
- HUD：剩余步数 chip、分数 + 星级进度条、目标分 chip；提示/重开/静音按钮。

### 限时制（规则/玩法变体，2026-07-19 验证）

- **delta**：限步制（30 步）→ **限时制（60 秒）**。`MOVES_MAX` → `TIME_MAX=60`；`G.moves` 从"剩余步数"改为"用步统计"（不再限量）；`update` 头部加限时心跳（时间每帧扣减）；达标即胜不变；判负从"步数耗尽"改为"时间归零且回到 idle"；胜利奖励从剩余步数 ×100 改为剩余时间 ×50；HUD 左 chip 从步数改为剩余秒数（≤10s 警示）；标题/结算文案同步。棋盘/消除/特殊棋子零改动。
- **红线重推**：
  - 目标分红线（限时版）：实测 bot 11 步 ≈19.2s 模拟用时得分（去奖励）3720 → 秒均 ≈194 分/s；普通玩家按 1/3 难度带折算 ≈60~100 分/s → 60s 目标 3000 贴下限可达（= bot 容量 11640 的 26%，过红线 2 校验）；
  - 时长红线：60s 经典档——太短连锁引擎展不开，太长紧张感消失；
  - **结算时机红线（变体陷阱）**：时间归零必须等当前连锁结算完（相位回 idle）再判胜负，否则连锁中的分被没收；
  - **bot 验收口径同步**：`bot_match3.js` 加制式探测（`G.timeLeft` 存在即限时模式），MOVES_MAX 移出必备符号、步数上限断言按制式分叉——**改胜负制式必须同步改 bot 口径，否则 bot 误报**。
- **验证证据**：变体 `WIN: true`（11 步、剩余 40.8s、总分 5770 含奖励）；基线回归 `WIN: true`（bot 改动未影响限步模式）；chips `audit_ui_align.py` PASS（0.0px）；HUD 前后对比图经用户验收（2026-07-19）。
